Multicolour branding brief

Multicolour is an open source NodeJS library that generates a REST API, doing as much of the repetitive work for you so that you get straight to the interesting solutions for your clients. All you need to do is write JSON.

I (Dave Mackintosh) started building Multicolour a little over a year ago, it evolved out of a pet project I'd used for best part of a decade to speed up frontend and app development.

I wanted to give back to the kind community that made my career what it is, I drew up a logo and hit the code with the message that your work shouldn't have to be repetitive and you can generate much of the work you do in just a few seconds.

While the logo was functional at conveying a technical approach to a technical problem, the tool itself has evolved into much more and so the brand doesn't convey the same message.

  1. Simple

The brand name is "Get Multicolour", the "get" shouldn't be ommitted in any way since there is less likely to be any kind of intellectual property infringement with it but as the current brand expresses, it's not the key to the brand name.

The current colour pallete can be binned (please, by all means do) I just really liked the 80's gradient and "night" like colours since it was an evening project for me.
