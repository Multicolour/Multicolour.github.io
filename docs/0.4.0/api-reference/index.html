Multicolour aims to make your life simpler, smarter and quicker. Instead of CRUD creation for each client and project, you can get straight into the interesting business logic.

Beneath Multicolour is a simple API that ties everything together.

Multicolour

The main Multicolour module is a message bus, using the Talkie package.

This allows Multicolour to act as two things:

  • An EventEmitter
  • a request and reply interface

The difference between the two is merely semantic, it allows us to make reading the code more natural and simplify certain aspects of the code resolution process ("What does this do?" .. "Oh, that.")

Multicolour "extends" all plugins with it's message bus so everyone works the same.

new Multicolour([config])

Create an instance of Multicolour, config is optional but should be a valid configuration object for multicolour to scan for blueprints and connect to your databases.

Returns: Multicolour
Example:

const Multicolour = require("multicolour")

const my_service = new Multicolour({
  content: `${__dirname}/content`
})

multicolour.get(key String)

Get any of the following properties values from Multicolour core, some properties are only available once the service has been .start()-ed.

  • "cli" - Object Uninitialised CLI module.
  • "config" - Map The config as a Map, first level properties are also Maps.
  • "stashes" - Map Namespaced storage for sundrie objects.
  • "env" - String the process.env.NODE_ENV value at startup.
  • "has_scanned" - Boolean whether Multicolour has .scan()ned for blueprints.
  • "package" - Object The contents of the app's package.json if any.
  • "blueprints" - Object Only available after .scan() has run.
  • "database" - Map Only available after .start() has completed, the database plugin.
  • "server" - Object Only available after .start() has completed, the server plugin.

Returns: Any
Example:

console.log(my_service.get("env")) // development

multicolour.request(key)

Request a property from Multicolour, this is merely semantic sugar. Available properties are:

  • "cli" - CLI newly initialised CLI module.
  • "new_uuid" - String a unique v4 uuid.
  • "storage" - Get the current storage plugin.

Returns: Any
Example:

console.log(my_service.request("new_uuid")) // 5d31d224-bf23-49ed-9db0-7a50ab924f4f

multicolour.scan()

Tells Multicolour that you and the server are ready to be scanned for blueprints. The search path is dictated by the config.content property.

Once scan has completed a new property is made available via the Multicolour.get() interface. These are

  • "blueprints"

Source Code
Returns: Multicolour
Example:

const my_service = require("multicolour")
  .new_from_config_file_path("./config.js")
  .scan()

multicolour.use(Plugin)

Tell Multicolour to register and "use" the supplied plugin. You plugin must have a register method on it.

Source Code
Returns: Multicolour
Example:

class my_plugin {
  register(Multicolour) {
    Multicolour.reply("hello", "world")
  }
}

const my_service = require("multicolour")
  .new_from_config_file_path("./config.js")
  .scan()

  .use(my_plugin)

console.log(my_service.request("hello")) // world

multicolour.start(done)

Start the database server(s) and the http server and start listening for requests. When both services are ready done is called.

Triggers a server_starting event on the Multicolour instance. Also sets up a listener for SIGINT on the process so the services can be .stop()ped gracefully

Source Code
Returns: Multicolour
Example:

require("multicolour")
  .new_from_config_file_path("./config.js")
  .scan()
  .start(() => console.log("Database server started"))

multicolour.stop(done)

Stop the database server(s) and the http server and stop listening for requests. When both services are stopped done is called.

Triggers a server_stopping event on the Multicolour instance.

Source Code
Returns: Multicolour
Example:

const my_service = require("multicolour")
  .new_from_config_file_path("./config.js")
  .scan()

setImmediate(() =>
  my_service.start(() => console.log("Database server(s) started")))

my_service.stop(() => console.log("Database server(s) stopped"))
