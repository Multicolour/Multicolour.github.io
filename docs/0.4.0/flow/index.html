Flow

Flows are a simple way to test the way your API functionality in a way your user or developer might interact with it.

These are called flows, as the user flows through the API to perform their desired actions, it's our job as developers to ensure it all goes smoothly.

Getting started with Multicolour Flows

There are two ways to create your flows, the Multicolour example app exports our service in it's app.js file and we simply pass that to the Flow constructor.

I.E

// tests/simple-flow.js
"use strict"

const Flow = require("multicolour/flow")
const service = require("../app")

new Flow(service)
  .create("person", {
    name: "Multicolour Flow",
    age: 1,
    secret: "super private awesome secret"
  })
  .read("person", 1)
  .create("gadget", {
    type: "laptop",
    cost: 1200,
    brand: "Apple",
    owner: "1"
  })
  .read("gadget", 1)

  // Run the flow.
  .run()

The alternative way to write your flows is to use the .flow function on Multicolour

API

Flows should read easily, they're written by developers for developers but they should still resemble our native language.

Inside of multicolour/flow/lex.js you will see a list of the terminologies that Flows currently understand and translates them to HTTP verbs.

I.E

  • "create" is turned into "post"
  • "update or create" is turned into "put"
  • "destroy" is turned into "delete"

So on and so-forth, this lets us write expressive tests in an easy to understand way. There are several helper functions that go to aid this as well documented below.

Where search_payload appears as an argument to any of the below functions, where you are only going to search based on id can omit the usage of an object. I.E change { id: 1 } into simply just 1 and Flow will make it an object for you.

.create(String model_name, Object payload)

Creates a new Task to create a new row on the model_name collection with the payload values.

.read(String model_name, Object search_payload)

Creates a new Task to read a row on the model_name.

If an id attribute is present within search_payload it is plucked from the search payload and appended to the url, for instance.

.read("gadget", { id: 1, type: "tablet" })

Will be turned into a get request to this uri:

/gadget/1?type=tablet

.update(String model_name, Object search_payload, Object payload)

Creates a new Task to update a row on the model_name collection based on the search_payload with the payload values.

.delete(String model_name, Object search_payload)

Creates a new Task to delete a row on the model_name collection based on the search_payload.

.then(String lexable_verb, String model_name, [Object search_payload, Object payload])

.then is an expressive helper function that allows any of the above documented functions to run using any of the "lex-able" tokens found in the lexicon

Example usage of .then

my_flow.then(
    // Update or create a row
    "update or create",

    // In the "gadget" collection.
    "gadget",

    // Either find or create a row with id: 2
    { id: 2 },

    // And these values.
    { name: "https://getmulticolour.com" })

.run()

Simply runs your flow as a test suite.
